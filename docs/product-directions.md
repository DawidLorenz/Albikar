# Albikar - Product Directions

This document defines concrete products that can be delivered on top of Albikar.

## 1) Small 3D game engine (Unity-lite direction)

Target features:
- PBR (metal/roughness), HDR, tone mapping, bloom.
- Scene graph + ECS + scene editor.
- glTF 2.0 import (meshes/materials/skeletal animation).
- Prefabs, component system, scene serialization (JSON/binary).
- Hot-reload for shaders and assets.

Delivery result:
- Production of small 3D games (FPS/TPP/platformer), cinematic demos, visualization tools.

## 2) Rendering engine / graphics sandbox (portfolio direction)

Target features:
- Deferred or Forward+ rendering.
- SSAO/SSGI, soft shadows (PCF/PCSS), TAA.
- GPU-driven rendering (culling + indirect draw).
- Frame graph and render passes.
- Debug views (normals/roughness/overdraw).

Delivery result:
- Strong graphics-programming portfolio project.

## 3) 2D/2.5D engine (faster-to-market direction)

Target features:
- Sprite batching, atlases, text, particles.
- 2D physics, tilemaps, 2D lighting.
- Level editor.

Delivery result:
- Faster delivery path for indie game production.

## 4) Simulation / visualization framework (professional tooling direction)

Target features:
- Data streaming, large scenes, LOD, instancing.
- Profiling dashboards and diagnostics.
- Deterministic tick and replay.

Delivery result:
- Tooling for simulation, digital twins, and training environments.
